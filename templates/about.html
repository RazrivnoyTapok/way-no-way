Проект

Главная страница

Сайт служит как социальная сеть для обмена файлами маршрутов, созданных в приложении. Зайдя на сайт, вы увидите главную страницу, где отображаются все посты с сортировкой от самых недавних к самым первым. На главной странице посты отображаются с неполной информацией, и чтобы увидеть информацию полностью, нужно перейти по гиперссылке в имени поста. Чтобы увидеть информацию об авторе, перейдите по гиперссылке в имени автора поста. На этих страницах информация расписана в виде таблиц.

Аккаунты

Чтобы постить свои маршруты на сайт, вам нужно зайти в существующий аккаунт или создать новый. Если вы не вошли в аккаунт, в левом верхнем углу будут 2 кнопки для регистрации и входа соответственно. При регистрации нужно нужно заполнить несколько полей: электронную почту, пароль, никнейм, остальные поля опциональные. После регистрации вас перенаправят на страницу входа в аккаунт, где нужно войти с электронной почтой и паролем. Изменить профиль можно, нажав на свой никнейм в левом верхнем углу (появится вместо кнопок), что направит вас в ваш личный кабинет. Там вы можете просмотреть собственный профиль, изменить его, выставить настройки сайта или увидеть собственные посты. При изменении профиля для оставления полей такими, какие они были, не ставьте в них ничего.

Посты

Если вы вошли в аккаунт, вы можете создать свой пост по соответствующей кнопке на полоске навигации сверху. Вас попросят ввести данные поста, в том числе прикрепить файлы, созданные в приложении (см. раздел "Приложение" на этой странице). После создания поста он сразу же будет отображаться на главной странице. Если пост приватный, то он будет виден только вам. В названии маршрута рекомендуем обозначать главные точки маршрута (например, начальную и конечную) или место маршрута (например, город или страну), чтобы пользователям было легко найти ваш пост по этим критериям.

Приложение

В приложении вы можете создать файл маршрута, который можно будет загрузить на сайт как пост. Чтобы загрузить приложение, нажмите на кнопку "Приложение" на навигационной полоске. Распакуйте архив, оба файла оставьте в одной папке. Запустите приложение. Откроется карта, по которой можно передвигаться, зажав левую кнопку мыши и двигая мышь. Покрутив колесо мыши, можно изменить масштаб показа карты.
Сверху есть кнопки для работы с файлами. Чтобы работать с ними, введите в нужное поле имя файла, который вы можете открыть в приложении или сохранить.
Справа есть кнопка "прицел", которая включает прицел для более удобной работы с линиями и метками.
Дальше расположены кнопки для работы с линиями на карте. Всего доступно 3 линии, которые можно переключать соответствующей кнопкой. Линия может иметь от 2 до 100 точек, при этом 3 линии в сумме должны иметь не более 100 точек. Для добавления точки или удаления последней точки нажмите соответствующие кнопки. Параметры линии можно указать в любой момент, главное не забыть после изменения линии нажать "сохранить настройки". Открывая файл, не забудьте нажать "сохранить настройки", чтобы все линии корректно отобразились!
Ещё ниже расположены кнопки для работы с метками. У меток можно настраивать стили, цвета, размер и контент, переключая соответствующие кнопки. Контент меток включает в себя числа от 1 до 100, в зависимости от стиля метки этот диапазон может меняться.
Под картой расположены кнопки для поиска. Место можно найти через верхнее поле ввода, а 3 ближайших к точке организации введённого типа - через нижнее. Полный адрес объекта будет отображаться между этими полями.
Маршрут сохраняется как 2 файла - картинка с карты и текстовый файл с информацией о маршруте (под одинаковым названием, в папке приложения).

API

К проекту подключен API на основе Flask-restful.
У API к пользователям есть страница /api/mappers.
При GET запросе к /api/mappers будет получен список всех пользователей с их nickname, info и join_date.
При GET запросе к /api/mappers/*id* будет получена та же информация, но об одном пользователе с нужным id.
При POST запросе к /api/mappers с нужными данными создастся аккаунт. В качестве параметров необходимо передать nickname (str); info (str); email (str); hashed_password (str).
При DELETE запросе к /api/mappers/*id* будет удалён пользователь с указанным id.
У API к постам есть страница /api/routes.
При GET запросе к /api/routes будет получен список всех постов с их name, description, author, creation_date, private, ll, z, l, info_r, info_m.
При GET запросе к /api/routes/*id* будет получена та же информация, но об одном посте с нужным id
При POST запросе к /api/routes с нужными данными создастся пост. В качестве параметров необходимо передать name (str); description (str); author (int), private (bool), ll (str), z (str), l (str), info_r (str), info_m (str).
При DELETE запросе к /api/mappers/*id* будет удалён пользователь с указанным id.
